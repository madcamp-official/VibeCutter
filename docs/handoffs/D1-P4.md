## 제공 인터페이스

### 1. inventory (`datasets/inventory.py`)
- 입력: `datasets/inventory.yaml`(또는 benchmark).
- 출력: `Inventory.apps: list[AppEntry]`, `.coverage()`,
  `AppEntry.to_target_stub() → {"id", "adapter"}`(P2 manifest / P1 Target 시드).
- 실패/예외: 계약 위반 시 `InventoryError`(로드 자체가 실패).

### 2. SAST candidate (`scanners/sast/semgrep_runner.py`)
- `run_semgrep(target_root, *, run_id, config="auto", ruleset_focus=None) → list[Candidate]`
- `parse_semgrep_output(data, *, run_id, ruleset_focus=None) → list[Candidate]`(순수 함수)
- 출력: `Candidate`. cwe←metadata.cwe(`CWE-89`), endpoint←None, source_symbols←`["<path>:<line>"]`,
  confidence←0.3~0.9, signals←`["semgrep:<rule>","severity:..","focus:<group>"]`.
  id `cand-sast-<sha1(run_id|rule|path|line)[:12]>`(결정적·dedup).
- 실패/예외: `SemgrepUnavailableError` / `FileNotFoundError` / `CalledProcessError`.

### 3. SCA candidate (`scanners/sca/osv_runner.py`)
- `run_osv(target_root, *, run_id) → list[Candidate]` / `parse_osv_output(data, *, run_id)`.
- Candidate: cwe←database_specific.cwe_ids, source_symbols←`["<lockfile>:<pkg>@<ver>"]`,
  signals←`["sca:osv","category:sca","vuln:<id>","pkg:..","ecosystem:..","severity:.."]`.
  **focus 태그 없음**(의존성은 3군과 별개). id `cand-sca-<sha1>`.
- 실패/예외: `OSVUnavailableError` / `FileNotFoundError` / `CalledProcessError`.

### 4. Baseline 하네스 (`eval/baseline.py`)
- `ground_truth_from_inventory(bench) → {app_id: set[focus]}`(expected_vulns 기반).
- `focus_set_from_candidates(cands) → set[focus]`(Candidate 의 `focus:` signal 수집).
- `evaluate(pred, truth) → BaselineReport`(overall+per-group precision/recall/FPR/F1/TPR-FPR).
- 평가 단위 = **(앱 × 3군) 셀**(공개앱은 앱 단위 라벨뿐). detector-agnostic.

### 5. 코드 인덱스 (`model/code_index.py`)
- `CodeIndex.build(root)`; `.search(query, k, embed_fn=None) → list[SearchHit]`;
  `.find_symbol(name) → list[Symbol]`. BM25 기본, `embed_fn` 훅으로 임베딩 교체(GPU 선택).
- **P3 root-cause locator(7.4절)가 소비**.

### 6. 전 앱 스캔 배치 (`scanners/batch_scan.py`)
- `run_batch(apps, workdir, *, clone_fn, scan_fn) → list[BatchItemResult]`(clone/scan 주입 가능).
- CLI: `python -m scanners.batch_scan --workdir <dir> [--sca] [--dry-run]`.
- 출력: `<workdir>/candidates/<app>.candidates.jsonl` + `summary.json`. 앱 단위 실패 격리.

## 검증

- 실행한 테스트: 5개 스위트 **총 36 pass**(sast9/sca9/batch4/baseline6/code_index8).
  `python -m datasets.inventory` → 41개 요약·3군 커버리지 정상. baseline 을 실제 벤치마크
  inventory 정답에 연결(15앱·45셀) 확인.
- **통합(rebase 후) 확인**:
  - `contracts.schemas.Candidate` 스키마 미변경 → 내 SAST/SCA/baseline 완전 호환, 36 pass 유지.
  - **inventory id 계약 유지**: P2 `targets/compose/*.yaml` 16개 id 가 전부 내 inventory
    41개에 포함(P2 에만 있는 id 0개). 내 잉여 25개는 P2 미착수분.
  - `scanners/` 는 P3(`playwright.py`)와 **공유 폴더**. 파일명 겹침 없어 공존. 내
    `scanners/__init__.py`(docstring)는 rebase 에서 P3 의 빈 버전으로 덮였으나 import 는
    `__init__` 내용에 의존 안 해 무해.
  - 충돌 마커 0건, 내 파일 전부 untracked.
- 아직 실제 환경에서 검증 못 함: **semgrep/osv-scanner 실주행 미수행**(미설치). 파서는
  실도구 `--json` 포맷 모방 fixture 로만 검증 — 밤 배치에서 실결과 1회 대조 필요(fixture
  제거 조건). code_index 임베딩 백엔드는 훅만(실제 임베딩 모델 GPU 확보 후). inventory
  stack/adapter 는 P2 build/health 확인 전까지 `verify=true`.

## 다른 역할에 필요한 사항

- **P1**: `vc_run_sast`(mcp_server/tools_analysis.py, 스텁) 배선 시
  `scanners.sast.run_semgrep(target_root, run_id=run_id)` 호출 → `Candidate[]`. SCA 는
  `scanners.sca.run_osv(...)`. 밤 배치 산출물 `<workdir>/candidates/<app>.candidates.jsonl`
  + `summary.json` 을 evidence/candidate store 로 흡수할 포맷만 확정해달라.
- **P2**: (a) `python -m datasets.inventory --targets` 의 `{id, adapter}` 와 P2 manifest 의
  `adapter` 값을 **대조**해달라(내 adapter 는 제안값). (b) 이미 소스를 clone 하는 경로
  (`.vibecutter/targets/sources/<id>`)를 알려주면 `batch_scan` 이 **재-clone 없이 그 경로를
  스캔**하도록 맞추겠다(중복 clone 방지). (c) inventory(41 카탈로그) vs targets/(16 manifest)
  **단일 진실 소스**를 무엇으로 할지 합의 필요.
- **P3**: SAST Candidate 의 `endpoint`=None(route mapper 소유) → route/role 매핑으로 채워야
  IDOR 검증이 붙는다. `signals` 의 `focus:<group>` 로 3군 필터. root-cause locator 는
  `model.code_index`(`CodeIndex.build(root).search(query)`/`.find_symbol(name)`) 소비.
  `scanners/playwright.py`(P3) 와 내 scanners 는 같은 `Candidate` 계약을 공유(동적 vs 정적).

## 결정·가정·리스크

- **[리스크] requirements.txt 에서 `semgrep` 이 merge 중 빠짐**. 현재 `PyYAML==6.0.3`(내
  yaml 사용은 정상), `sqlmodel` 만 있고 `semgrep` 없음. **SAST 실행하려면 `semgrep` 재추가
  필요**(공유 파일이라 다른 역할과 머지 조율).
- **[리스크] `scanners/` 공유 + 커밋 범위**: `git add scanners/` 로 뭉치지 말고 내 파일만
  (`scanners/sast scanners/sca scanners/batch_scan.py scanners/test_batch_scan.py`) add — P3
  파일 오염 방지.
- **[가정] focus 추론은 CWE 번호 우선**(CWE-79=xss, 89=injection, 639/862/863=idor…),
  rule_id/category 폴백. OWASP "A03 Injection" 범주는 XSS 포함해 거칠어서 미사용(초기 버그
  수정 이력). secret 등 3군 밖 finding 은 Candidate 로 나오되 `focus:` 태그 없음(의도).
- **[가정] inventory 41개는 미감사 자동 분류**: repo_url/stack/adapter 정확도는 P2 검증
  전까지 미보장. 12.4 확장 성공(30+ inventory) 충족, 목표 성공(10~15 실행)은 P2 build 후 확정.
- **[관례 불일치] 테스트**: 팀은 top-level `tests/` + pytest, P4 는 모듈 옆 `test_*.py` +
  자체 러너(`python -m ...`). 충돌 아님. 통합 시 pytest 로 수렴할지 D2 논의.
- **[배치 산출물]** workdir 은 `runs/` 아래로 둘 것(`.gitignore` 가 `runs/` 무시). 그 외
  경로면 clone·candidate 가 커밋될 수 있음.
